# すたログ 📚

科目を選んで再生▶ / 停止■ するだけの、勉強記録アプリ(一般公開用)。

デザイン方針: 淡いパステルレインボー基調のジェンダーニュートラル(特定の性別に寄せない)。丸ゴシック(Zen Maru Gothic)のやさしい雰囲気は維持。絵文字は星・動物・天気・食べ物系のみ(リボン/ハート/花は使わない)。文字色はWCAG AA(コントラスト比4.5:1以上)を確保: 補足文字 `--text-soft: #756d87`(4.9:1)、アクセント文字 `--accent-deep: #5661c9`(5.3:1)。guide.html も同値。

- 本体は `index.html` 単一ファイル(CSS/JS埋め込み、サインイン不要、サーバー不要)。付属ページ・PWA用ファイル:
  - `guide.html` — 使い方ページ(アプリのフッターからリンク。AdSense審査用のコンテンツも兼ねる。コラム類は信ぴょう性担保できないため置かない方針)
  - `manifest.json` — アプリ名・テーマ色・アイコン定義
  - `sw.js` — Service Worker(オフライン起動対応。本体はネット優先で更新が届く)
  - `icon-180.png`(iOS用)/ `icon-192.png` / `icon-512.png` / `icon-maskable-512.png`(Android用)
  - `icon.svg` / `icon-maskable.svg` — アイコンの元データ(にじ+にこにこ星+雲+「すたログ」の文字。文字はZen Maru Gothic、Webフォント読込待ちに `--virtual-time-budget=10000` が必要)
  - PNG再生成手順: ヘッドレスChromeで 512px を撮る(`chrome --headless=new --screenshot=icon-512.png --window-size=512,512 --virtual-time-budget=10000 icon.svg`)→ 180/192 は PowerShell(System.Drawing)で 512px から縮小(Chromeは小さいwindow-sizeだと正しく描画されない)
- `ogp.png` — SNSシェア用カード画像(1200×630、OGPタグから参照。デプロイ対象)
- `ads/` — SNS広告用画像(正方形1080/ストーリーズ1080×1920 の PNG+SVG元データ。広告入稿用でデプロイ不要)
- デプロイ時は上記すべてを同じフォルダに置く(icon.svg・ads/ は任意)
- `lp.html` — SNS広告の遷移先ランディングページ(広告の各特長スライドに対応するセクション+CTA。CTAクリックは gtag イベント lp_cta_click で計測。デプロイ対象)
- `robots.txt` / `sitemap.xml` — SEO用(デプロイ対象)
- SEO: title/description/keywords にキーワード(勉強・記録・ログ・アプリ・簡単・シンプル等)、canonical、JSON-LD(index=WebApplication、guide=FAQPage)、h1=ロゴ。head の Google タグ(gtag/AdSense)はユーザー管理につき削除しないこと
- ドメイン移行時の注意: index.html / guide.html の og:url・og:image・canonical・JSON-LD内URL、sitemap.xml・robots.txt のURLを新ドメインに書き換えること
- 記録はブラウザの localStorage(キー `stalog.v1`、データ version 2)に保存。履歴は約13ヶ月(400日)保持 — 受験1年分を想定
- 想定用途: 紙の「合格手帳」の日々の記録への手書き転記元(「きょう」タブの時系列をそのまま書き写せる)

## 機能

- **科目制**: プリセット(国語・数学・英語・理科・社会)+ ユーザー追加/削除可(科目ごとに色つき)。加えて任意の「メモ」(何をやるか)を記録できる
- **ユニバーサルカラー切替**: 「いろの設定」のトグルで科目色を、東京都カラーユニバーサルデザインガイドライン掲載の「CUD推奨配色セット」(東京大学 伊藤啓准教授)のアクセントカラー8色に切替(黄色は白背景と見分けにくいというガイドラインの注意に従い除外。国語=赤/数学=青/英語=紫/理科=緑/社会=オレンジ)。`subjectColor()` が一元管理しているので、チップ・内訳・ドーナツ・記録ドットすべてに反映。設定は `state.cud` として保存
- **手動記録・修正**: 「✍️ 手動で記録をつける」で過去にさかのぼって記録を追加。既存の記録はタップで編集(科目/メモ/日付/開始/終了)。時刻が未変更なら元の秒を保持。他の記録との時間重複は許容(チェックしない)。終了<開始のときは確認のうえ夜またぎ(翌日終了)として保存
- **記録をみる**:
  - 「きょう」タブ — 手帳転記用の時系列リスト(開始/終了時刻・科目・メモ・時間)
  - 「過去」タブ — 期間指定(クイック 7日/30日/ぜんぶ + 日付指定)で合計時間・1日平均・科目別割合ドーナツグラフ・日別棒グラフ(62日超は週別に自動切替)・日付グループ履歴
- **バックアップ**: 「バックアップする」(JSON、UIでは JSON という言葉は出さない)+「復元する」
  - スマホ(iPhone / Android)では Web Share API で共有シートが開き、「ファイルに保存」「Drive」「LINE」などに保存できる
  - 共有シート非対応の環境(PC等)は通常のファイルダウンロードに自動フォールバック
  - 復元は記録 id で重複排除(何度読み込んでも安全)、科目リストも同名スキップで取り込む

> ⚠️ このフォルダは社内ツール Loggy(`D:\projects\loggy`)とは完全に別プロジェクト。相互にファイルを共有しない。

## ホーム画面に追加(PWA)

HTTPS でホスティングした URL を直接開くと、スマホのホーム画面にアプリとして追加できる:

- **iPhone (Safari)**: 共有ボタン →「ホーム画面に追加」→ レインボーアイコンで全画面アプリとして起動
- **Android (Chrome)**: メニュー →「アプリをインストール」(または「ホーム画面に追加」)
- 一度開けばオフラインでも起動できる(Service Worker がキャッシュ)
- ⚠️ Google サイトの埋め込み iframe からはインストールできない。サイトに「アプリとして使う」ボタン(直接 URL へのリンク)を置いて誘導すること

## Google サイトへの埋め込み手順

1. `index.html` をどこかにホスティングして URL を用意する
   - 例: GitHub Pages(無料)… 公開リポジトリに `index.html` を置き、Settings → Pages で公開
   - Google サイトの「埋め込みコード」に HTML を直接貼る方法もあるが、サイズ上限があるため URL 埋め込み推奨
2. Google サイトの編集画面 → 「挿入」→「埋め込む」→「URL」タブに公開 URL を貼り付け
3. 埋め込みブロックのサイズを **幅いっぱい × 高さ 700px 以上** に調整(アプリは 480px 幅基準のモバイルファースト)
4. サイトを「公開」→ 公開設定を「ウェブ上の誰でも」にする

### 注意

- Google サイト埋め込み(sandbox iframe)では、ブラウザ設定によって localStorage が分離・削除されることがある
  → アプリ内のバックアップ(CSV/JSON)案内で対応済み
- 埋め込み iframe では共有シート(Web Share)やダウンロードがブロックされる可能性がある
  → その場合はサイトに「アプリを直接開く」リンク(ホスティング先 URL)も併記しておくと確実
- 新 Google サイトは AdSense 非対応。収益化する場合は独自ドメイン + 通常ホスティングに移行する
